What is SQL Injection?

SQL Injection (SQLi)

SQL Injection (SQLi) happens when an app inserts user input directly into a database query without proper checks. This lets attackers manipulate the query, often to bypass login, steal or change data, or take over the system. It's common in login forms, search bars, URL parameters, and any input tied to a database.

How Does It Work?

Understanding the mechanics behind SQL Injection attacks.

Vulnerable Query Example

SELECT * FROM users WHERE username = 'input' AND password = 'input';

This is a typical login query that directly concatenates user input into the SQL statement.

Malicious Input

admin' OR '1'='1

When this input is inserted, it manipulates the query logic.

Resulting Query

SELECT * FROM users WHERE username = 'admin' OR '1'='1' AND password = 'input';

The OR '1'='1' condition is always true, bypassing authentication.

How to Fix SQL Injection

Best practices and mitigation strategies for preventing SQL Injection attacks.

Use Parameterized Queries

Never concatenate user inputs into SQL. Always use prepared statements or ORM tools that separate code from data.

Safe Query Example:

$stmt = $pdo->prepare("SELECT * FROM users WHERE username = :user AND password = :pass");
$stmt->execute([':user' => $username, ':pass' => $password]);

Validate Inputs

Restrict input length and allowed characters based on expected data types and formats.

Principle of Least Privilege

Ensure database accounts used by your app have only necessary permissions, not admin access.
